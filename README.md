# Needleman-Wunsch Benchmarking Framework

This project compares four C99 implementations of global sequence alignment:

- Sequential baseline
- Pthreads wavefront implementation
- OpenMP wavefront implementation
- MPI master-worker task farm implementation

The benchmark framework generates reproducible DNA sequence inputs, runs each implementation, saves raw program output, and writes a CSV file with timing, speedup, and efficiency.

## Directory Structure

```text
.
├── data/
│   ├── input_generator.c
│   └── generated/              # Created by the experiment script
├── scripts/
│   └── run_experiments.sh
├── results/
│   ├── benchmark_results.csv   # Created by the experiment script
│   └── raw_outputs/            # Full output from every run
├── src/
│   ├── sequential/main_seq.c
│   ├── pthreads/main_pth.c
│   ├── openmp/main_omp.c
│   └── mpi/main_mpi.c
└── Makefile
```

## Dependencies

Required:

- GCC or another C99 compiler
- GNU Make
- Bash
- AWK

Optional, depending on implementation:

- POSIX Pthreads support for `main_pth`
- OpenMP support for `main_omp`
- MPI implementation such as OpenMPI or MPICH for `main_mpi`

On Ubuntu/Debian, a typical setup is:

```bash
sudo apt update
sudo apt install build-essential openmpi-bin libopenmpi-dev
```

## Input Generation

The generator creates random DNA sequences using the alphabet:

```text
A C G T
```

Supported lengths:

```text
100, 500, 1000, 5000, 10000, 50000
```

Manual usage:

```bash
make input_generator
mkdir -p data/generated
./bin/input_generator data/generated 3 2026
```

Arguments:

```text
./bin/input_generator <output_dir> <pairs_per_size> [seed]
```

Each generated file has this format:

```text
<number_of_pairs>
<sequence_A_pair_0>
<sequence_B_pair_0>
<sequence_A_pair_1>
<sequence_B_pair_1>
...
```

This format can be passed directly to the MPI program. The benchmark script also uses the same files for sequential, Pthreads, and OpenMP runs by feeding each pair separately.

## Running Experiments

Run the full benchmark:

```bash
bash scripts/run_experiments.sh
```

Optional environment variables:

```bash
PAIRS_PER_SIZE=5 SEED=1234 bash scripts/run_experiments.sh
```

The script automatically attempts to compile:

- Sequential
- Pthreads
- OpenMP
- MPI
- Input generator

It runs worker counts:

```text
1, 2, 4, 8
```

For Pthreads and OpenMP these are thread counts. For MPI these are process counts.

## Output Files

Raw program output is saved in:

```text
results/raw_outputs/
```

The summary CSV is saved as:

```text
results/benchmark_results.csv
```

CSV columns:

```text
Algorithm,Input size,Threads,Processes,Execution time,Speedup,Efficiency
```

Definitions:

- `Execution time`: total measured runtime for the generated pairs of that input size.
- `Speedup`: sequential time divided by parallel time.
- `Efficiency`: speedup divided by thread or process count.

## Manual Execution Examples

Sequential:

```bash
printf "ACGT\nACGT\n" | ./bin/main_seq
```

Pthreads:

```bash
printf "ACGT\nACGT\n" | ./bin/main_pth --threads 4
```

OpenMP:

```bash
printf "ACGT\nACGT\n" | OMP_NUM_THREADS=4 ./bin/main_omp
```

MPI:

```bash
mpirun -np 4 ./bin/main_mpi < data/generated/input_len_100_pairs_3.txt
```

## Expected Outputs

For:

```text
ACGT
ACGT
```

Expected score:

```text
Alignment score: 4
Aligned sequence A: ACGT
Aligned sequence B: ACGT
```

For:

```text
GATTACA
GCATGCG
```

Expected score:

```text
Alignment score: 0
```

The exact alignment may differ only if an implementation changes tie-breaking. In this project, all implementations preserve the same tie-breaking policy as the sequential baseline.

## Reproducibility

This framework supports university reproducibility requirements in several ways:

- Inputs are generated by source-controlled C code.
- The random seed is recorded and configurable.
- Every implementation reads the same generated sequence pairs.
- Raw outputs are preserved for inspection.
- Summary metrics are written to a CSV file.
- Build commands are centralized in the Makefile.
- Thread and process counts are fixed by the experiment script.
- The sequential baseline is used consistently for speedup calculations.

To reproduce a run, record:

- Git/project version
- Compiler version
- CPU and memory information
- Operating system
- `PAIRS_PER_SIZE`
- `SEED`
- Thread/process counts
- Final `results/benchmark_results.csv`

## Troubleshooting

`make: command not found`

Install GNU Make or run the equivalent compiler commands manually.

`mpicc: command not found`

Install OpenMPI or MPICH. MPI benchmarks will be skipped by the script if the MPI target cannot compile.

`mpirun: command not found`

Install an MPI runtime or use `mpiexec` if available.

OpenMP linking fails with missing pthread symbols

Use a GCC build with OpenMP runtime support installed. Some Windows MinGW setups need additional runtime libraries.

Benchmarks take too long

Reduce the number of pairs:

```bash
PAIRS_PER_SIZE=1 bash scripts/run_experiments.sh
```

Very large inputs run out of memory

Needleman-Wunsch stores a full `(m+1) x (n+1)` matrix. Length `50000` can require enormous memory. Use this size only on machines with sufficient RAM, or reduce the size list in `scripts/run_experiments.sh`.

## Larger-Scale Benchmarking Improvements

Useful next steps:

- Add repeated trials and report mean, median, and standard deviation.
- Pin threads/processes to cores for less noisy timing.
- Record compiler flags and CPU details automatically.
- Split very large MPI inputs across files to avoid rank 0 reading everything first.
- Add correctness validation that compares every parallel score with sequential.
- Generate plots from the CSV using Python, R, or gnuplot.
- Use a job scheduler script for clusters, such as SLURM.
- Add memory usage tracking for large DP matrices.
- Benchmark weak scaling and strong scaling separately.
